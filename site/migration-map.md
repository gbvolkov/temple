# Карта миграции MODX → новая CMS

> Актуальная количественная готовность и незакрытые барьеры: `outputs/migration-readiness.md`. Автоматический импорт не публикует материалы и не перезаписывает записи, уже проверенные редактором.

Текущая CMS используется только как источник чтения. Миграция выполняется экспортом/скриптами и не требует редактирования исходных ресурсов.

| Источник MODX | Текущее представление | Новая сущность | Правило преобразования |
|---|---|---|---|
| Шаблон «Внутренняя» | `pagetitle`, `introtext`, TinyMCE `ta`, TV `image` | `page` или `news` | Определить тип по родителю и шаблону; HTML очистить и разбить на смысловые блоки |
| «Новости прихода» | дочерние ресурсы и Ditto | `news` | Сохранить даты, заголовок, текст, изображения и порядок; назначить рубрику |
| «Анонсы» | дочерние ресурсы | `news` с датами события | Извлечь дату события отдельно от даты публикации |
| «Расписание богослужений» ID 14 | большой HTML в TinyMCE | `service[]` | Разобрать строки по дате/времени; неполные записи отправить на ручную проверку |
| «Расписание занятий» ID 48 | большой HTML в TinyMCE | структурированное расписание `parish_section` | Выделить группы, время, предмет и преподавателя |
| EvoGallery / шаблоны «Галерея», «Альбом» | ресурсы по годам и фото | `gallery` + `photos[]` | Год становится фильтром, событие — альбомом; сохранить подписи и порядок |
| «Приходской листок» ID 20 | годы-ресурсы + multiTV `prlist` | `leaflet_issue` | Каждая строка multiTV становится отдельным выпуском с номером, PDF и обложкой |
| Главная ID 1, TV `soc` | JSON/multiTV социальных ссылок | `site_contact.social_links` | Нормализовать сеть, URL и активность; отключить устаревшие ссылки |
| Главная ID 1, TV `homeslider` | JSON/multiTV с изображениями/текстом | `news.featured` / `site_notice` | Сопоставить существующие слайды с материалами; убрать несвязанные баннеры |
| Духовенство | список и дочерние биографии | `clergy` | Разделить сан, имя, должность, фото, биографию, день ангела и порядок |
| Направления прихода | страницы-списки с дочерними записями | `parish_section` + связанные `news/gallery` | Статическое описание — в направление, события — в новости и альбомы |
| Трансляции | текстовые ссылки YouTube | `video` | Нормализовать URL, название, дату, категорию и признак прямого эфира |
| Контакты ID 7 | HTML-текст | singleton `site_contact` | Разделить адрес, метро, телефон, email, часы, реквизиты и карту |

## Обязательная проверка каждого пакета

1. Количество исходных и импортированных материалов совпадает.
2. Дата публикации и дата события не перепутаны.
3. Все изображения и PDF доступны и имеют контрольную сумму.
4. Старый URL отвечает 301-редиректом на конкретный новый материал.
5. Выборка не менее 10 материалов каждого типа сверена визуально.
6. После выборочной проверки выполняется полный автоматический обход URL.

## Потенциальные проблемы исходных данных

- ошибочные относительные ссылки, которые создают пути вида `/o-hrame/o-hrame/...`;
- смешение HTTP и HTTPS;
- имена файлов с пробелами, кириллицей и неодинаковой кодировкой;
- изображения без alt-текста;
- даты, записанные внутри заголовка или HTML;
- дубли ресурсов и служебные страницы в одном дереве с публичным контентом;
- устаревшие или неработающие внешние ссылки.
