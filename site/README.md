# Интерфейс нового сайта и CMS

## Состав

- `index.html` — адаптивный публичный интерфейс с hash-маршрутами и загрузкой опубликованных материалов из API.
- `cms.html` — рабочий интерфейс редактора новой CMS.
- `styles.css`, `app.js` — публичный интерфейс и взаимодействия.
- `cms.css`, `cms.js` — интерфейс редактора и локальный предпросмотр.
- `cms-schema.json` — реализационная модель контентных типов, ролей и workflow.
- `migration-map.md` — сопоставление текущего MODX с новой CMS.
- `qa-inventory.txt` — проверяемые пользовательские сценарии и визуальные состояния.

## Локальный запуск полного приложения

Из корня проекта:

```powershell
uv run uvicorn server.app:app --host 127.0.0.1 --port 4174
```

Публичный сайт: `http://127.0.0.1:4174/`

Редактор CMS: `http://127.0.0.1:4174/cms.html`

## Важное ограничение

Новая CMS не подключена к текущему MODX и не изменяет его. Авторизованная панель MODX использовалась только для чтения и анализа структуры. Черновики и публикации сохраняются только в новой SQLite-базе.

## Проверка прототипа

- публичная часть: 8 разделов на 5 размерах экрана (40 проверок), без горизонтального переполнения и битых изображений;
- редактор: формы всех 9 типов материалов, вход, версии, загрузка медиа, черновик, предпросмотр, контролируемая публикация и панель миграции;
- контрольные размеры редактора: 1600, 1024, 768, 390 и 320 px;
- ошибки JavaScript и ошибки консоли браузера не обнаружены.

Если открыть папку `site` через простой статический сервер без API, интерфейс автоматически переходит в демонстрационный режим и использует только `localStorage`. При запуске FastAPI сохранение и публикация работают через новую CMS.
